# Protocole initial — ΔΩX

## Objet
Établir les règles minimales pour la gestion des artefacts, des audits et des contributions dans le projet ΔΩX.

## Règles de base
1. **Clarté des commits**  
   - Chaque modification doit être accompagnée d’un message de commit explicite.  
   - Les intentions peuvent être consignées dans `contextes/journal.md`.

2. **Protection des données sensibles**  
   - Les coordonnées réelles (`true_coords`) ne doivent jamais être commit en clair.  
   - Toute manipulation spatiale doit être documentée dans le journal.

3. **Audit et validation**  
   - Les audits simulés doivent être distingués des audits réels.  
   - Les résultats doivent être comparés et consignés dans `audit_spatial_v3.3.1-real.md`.

4. **Évolution du protocole**  
   - Ce document est évolutif.  
   - Toute modification doit être validée par consensus multi-agent (humain + IA).  
   - Les nouvelles règles doivent être ajoutées avec un identifiant (P002, P003…).

## Notes
Ce protocole sert de **socle minimal**.  
Il peut être enrichi au fil des versions (v3.3.2, v3.3.3…) pour intégrer de nouvelles pratiques, outils ou rituels.
